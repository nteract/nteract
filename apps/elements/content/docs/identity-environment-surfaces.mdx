Notebook UI needs product language for who is authenticated, who submitted work,
and what environment runs it. This page catalogs notebook-semantic composites,
not raw primitive components.

`NotebookIdentityBadge` and `NotebookIdentityGroup` wrap the shared Avatar
primitive with notebook actor/access facts. `EnvironmentSummary` renders the
shared typed environment projection for access, runtime, package details, sync,
and trust state. Environment runtime and package-manager icons live in
`src/components/environment` so notebook, settings, onboarding, and Elements use
one canonical set.

The fixtures come from `ElementsNotebookScenario`: desktop owner/read-only/
remote-room, public viewer, cloud editor/owner, credential-attention,
multi-operator, mixed-IdP, delegated submitter, runtime peer, system actor,
runtime-unavailable, and untrusted-dependency states. Desktop and cloud should
feed the same production components from their host adapters.

## Boundary

- Raw shadcn primitives stay implementation details.
- Actor labels come from notebook identity projections; access, runtime,
  package, sync, and trust facts feed the shared environment projection.
- Runtime and package-manager icons are shared environment primitives, not
  app-local notebook assets.
- Run attribution identifies the submitting actor or operator. It is not
  AI model lineage, and runtime peers are compute participants.
- Package info is notebook environment context, not an npm-style package profile.
