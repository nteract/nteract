`nteract/nteract` is the canonical home for notebook UI and rendering work.
This catalog is the in-repo successor direction for examples that used to live
around `nteract/elements`.

## What belongs here

- Runtime-free notebook UI fixtures.
- Rendering examples for markdown, rich output, and widgets.
- Component documentation for reusable notebook presentation surfaces.
- Migration notes for any old `nteract/elements` examples worth recreating.

## What stays elsewhere

This is not the general product docs site and it is not the architecture docs
archive. Existing architecture and design documents should remain under the
repo-level `docs/` tree.

## Current slice

Start with the outline rail and the first production-backed component surfaces.
The catalog should keep fixture-backed examples close to real notebook imports
without importing the desktop runtime, sync engine, or generated WASM bindings.

## Next slice

which surfaces are nteract-owned notebook concepts and which surfaces are still
generic shadcn primitives under the hood.
