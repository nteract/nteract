# Task 04: nono process supervisor

## Framing

Build a typed Rust supervisor that owns the lifecycle of a `nono` subprocess and the kernel grandchild it spawns. The empirically-confirmed quirk that drives this task: `SIGKILL` on the nono PID does **not** kill the kernel — the kernel reparents to init and survives. The daemon must independently track and signal **both** PIDs.

This task delivers the `nono::Supervisor` type. It does not yet wire into kernel launch (task 07). It depends only on task 01 (binary discovery).

## Background

Empirical truths (from testing — see `nono-empirical-tests.md`):

- nono inherits the caller's process group; it does not start a new one
- nono spawns the target process as a regular child (the kernel becomes nono's child, daemon's grandchild)
- nono additionally spawns `/usr/bin/log stream` as a sibling helper for sandboxd denial monitoring
- `SIGKILL`ing nono does not propagate to the kernel; the kernel survives
- Session ID is auto-generated by nono and is **not** controllable via flag (`--session-id` does not exist)
- Session ID appears on **stdout** (DEBUG line) only at `-vv`; the audit directory name is never printed

The supervisor must therefore:
1. Spawn `nono run -vv --profile <path> -- <kernel_argv>` in its own process group (so the daemon can clean up cleanly)
2. Monitor stdout, stderr, and exit code
3. Discover the kernel PID by reading nono's child list once nono has spawned it (use `pgrep -P <nono_pid>` semantics or the `procfs`/`sysctl` equivalent)
4. Track both PIDs independently
5. On shutdown, signal kernel first (clean shutdown), then nono
6. On unexpected nono exit, kill the kernel (orphan recovery)
7. On unexpected kernel exit, terminate nono (proxy is now useless)
8. Surface exit conditions through a typed result enum

## Technical steps

### 1. Module skeleton

Add `crates/runtimed/src/nono/supervisor.rs` (and re-export from `crates/runtimed/src/nono/mod.rs`).

Public API (define types in this task):

```rust
pub struct Supervisor {
    // private fields: handles, channels, tracked PIDs, state
}

pub struct SupervisorConfig {
    /// Argv for the kernel itself (the part that follows `--`).
    pub kernel_argv: Vec<OsString>,
    /// Path to the temp profile YAML produced by task 05. The supervisor does not
    /// generate it; it expects an already-written file.
    pub profile_path: PathBuf,
    /// Working directory for both nono and the kernel.
    pub cwd: PathBuf,
    /// Environment for the kernel (passed via nono).
    pub env: Vec<(OsString, OsString)>,
    /// Optional human label, passed as `--name` for audit log readability.
    pub name: Option<String>,
}

#[derive(Debug)]
pub struct SupervisorHandle {
    /// PID of the nono process itself (always present once spawned).
    pub nono_pid: u32,
    /// PID of the kernel grandchild. Populated once discovered; may briefly be None
    /// during the discovery race window.
    pub kernel_pid: Arc<AtomicI32>,
    /// Drain channel for stderr lines parsed off nono's stderr.
    pub stderr_lines: mpsc::Receiver<StderrLine>,
    /// Drain channel for stdout lines (DEBUG session ID lives here).
    pub stdout_lines: mpsc::Receiver<StdoutLine>,
    /// Resolves once nono exits.
    pub exit: oneshot::Receiver<SupervisorExit>,
}

#[derive(Debug, Clone)]
pub struct StderrLine {
    pub raw: String,
    pub at: Instant,
}

#[derive(Debug, Clone)]
pub struct StdoutLine {
    pub raw: String,
    pub at: Instant,
}

#[derive(Debug)]
pub enum SupervisorExit {
    /// nono exited cleanly with status 0
    CleanExit,
    /// nono exited with non-zero before kernel started (startup failure)
    StartupFailure {
        exit_code: Option<i32>,
        stderr_capture: Vec<String>,
    },
    /// nono exited unexpectedly while kernel was running
    ProxyDied {
        exit_code: Option<i32>,
        stderr_capture: Vec<String>,
    },
    /// Caller requested shutdown
    Shutdown,
}

impl Supervisor {
    pub async fn spawn(
        nono_binary: &Path,
        config: SupervisorConfig,
    ) -> Result<(Self, SupervisorHandle), SupervisorError>;

    /// Initiate graceful shutdown: SIGTERM kernel, then SIGTERM nono, then SIGKILL after timeout.
    pub async fn shutdown(self, grace: Duration) -> SupervisorExit;
}
```

### 2. Spawn nono correctly

- Argv: `["nono", "run", "-vv", "--profile", profile_path, "--name", name_or_default, "--", ...kernel_argv]`
- Use `tokio::process::Command` with `stdin = Stdio::null()`, `stdout = Stdio::piped()`, `stderr = Stdio::piped()`
- Set its own process group via `pre_exec(|| { libc::setpgid(0, 0) ... })` on Unix. This makes daemon-initiated `killpg` cleanup work without affecting nono's internal child handling.
- Pass `env` via `Command::envs` (nono inherits and forwards to the kernel)

### 3. Discover the kernel PID

After spawn, the kernel is somewhere in nono's child list. Discovery options:

- macOS: `sysctl` with `KERN_PROC_PPID` (the cleanest), or shell-out to `pgrep -P`
- Linux: read `/proc/<pid>/task/*/children` or `pgrep -P`

Pick a single Rust crate or implement a small helper that gives `children_of(pid: u32) -> Vec<u32>`. nono spawns at least two children (the kernel and the `log stream` helper). To distinguish:

- The kernel is the long-lived one matching the argv we passed; the `log stream` helper has `/usr/bin/log` as its executable
- Filter children by reading `comm`/argv of each candidate
- If multiple match, pick the one whose argv matches `kernel_argv[0]`
- Retry with backoff for up to ~2 seconds (the kernel may take a moment to spawn)

If discovery fails after the timeout, log a warning and proceed with `kernel_pid = None`. The supervisor still works for shutdown via signaling nono, but orphan-kernel detection is degraded.

### 4. Stream stdout/stderr

Spawn two background tasks:

- `stderr_drain_task`: read lines from nono's stderr, push `StderrLine` into the bounded channel (capacity ~2048; drop oldest on overflow with a counter for "lines dropped")
- `stdout_drain_task`: same for stdout

The drain tasks **must** keep reading even if no one consumes the channel (otherwise nono will block on a full pipe). When the channel is full, drop the oldest line and increment a counter logged once per second.

### 5. Watch both PIDs

Spawn a `process_watcher` task that:

- Awaits nono's exit via `Child::wait()`
- Concurrently polls (or uses `kqueue`/`pidfd` if you prefer) the kernel PID for liveness
- Whichever exits first triggers the appropriate `SupervisorExit` variant and signals the other

### 6. Shutdown semantics

`Supervisor::shutdown(grace)`:

1. Send `SIGTERM` to kernel PID (if known); else skip
2. Wait up to `grace / 2` for kernel exit
3. Send `SIGKILL` to kernel PID if still alive
4. Send `SIGTERM` to nono PID
5. Wait up to `grace / 2` for nono exit
6. Send `SIGKILL` to nono PID and `killpg(SIGKILL)` on its process group as a final fallback
7. Return `SupervisorExit::Shutdown`

The order matters: signaling the kernel first lets it flush; signaling nono after lets the audit log finalize cleanly.

### 7. Error types

```rust
#[derive(Debug, thiserror::Error)]
pub enum SupervisorError {
    #[error("nono binary not found: {0}")]
    BinaryNotFound(#[from] crate::nono::NonoUnavailable),
    #[error("failed to spawn nono: {0}")]
    Spawn(#[source] std::io::Error),
    #[error("kernel PID discovery timed out")]
    KernelDiscoveryTimeout,
}
```

### 8. Tests

- Spawn a no-op `nono run -vv --profile <empty> -- /bin/sleep 5` (use a temp profile that does not require credentials), confirm both PIDs are tracked, then `shutdown` and confirm both are dead.
- Empty/invalid profile → `StartupFailure`
- Manually kill the kernel PID externally → `process_watcher` produces `ProxyDied` (or its inverse) and triggers cleanup
- Stress test: drain task does not block when a consumer is slow

Skip these tests on Windows (nono is unsupported); gate via `#[cfg(unix)]`.
