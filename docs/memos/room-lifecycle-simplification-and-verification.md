# Room lifecycle: simplification and verification roadmap

The room lifecycle stack (source lifecycle, recovery journal, causal file checkpoints, reconciliation, progressive MCP connect) landed on main as stacked slices, and the tree still carries the landing scaffolding: superseded APIs kept by tests, `#[allow(dead_code)]` markers whose consumers have landed, cfg(test) wrappers preserving watcher-era code paths, duplicated staged/legacy logic, and comments narrating landing order instead of current invariants. A subsystem-by-subsystem mapping sweep, an adversarial verification pass on every proposal, and a cross-model review of the verification plans are now done. This memo is the merged output: a ranked simplification backlog (with rejected proposals recorded so nobody re-proposes them), and a ranked verification roadmap targeting the invariants that currently rest on comments, including the two incident classes that have actually shipped: the inotify IN_ACCESS watcher re-ingest loop (twice) and the generation-bump family.

### Consolidations

Each its own item, ranked by payoff per effort.

1. **Drop the watcher self-write time window, keep the content-hash skip** [M, ~18 LOC]. The layered-suppression simplification tied directly to both inotify IN_ACCESS incidents. Verify pass corrected the justification: `last_self_write` is stamped after the rename+commit, so the window never covered the gap it appears to guard; the manifest-fingerprint disjunct (set at journal-commit time under the coordinator lock) is what actually covers it, and the rename-to-commit micro-gap is handled by the pending-checkpoint comparison. The window also swallows genuine external edits within 600ms and can wedge autosave. Land with three tests first (see verification item 1), then delete the window, its constant, and the write-only `last_self_write` field.

2. **Collapse the serde and jiter .ipynb parsers** [L, ~95 LOC]. The serde parser is production code on the watcher path, not test scaffolding, so this is a behavior change that needs three divergences pinned before the switch: non-object cell entries (serde synthesizes a default cell, jiter drops it), missing `cells` key (serde None-skip vs jiter hard error; the watcher must map the new Err to skip-and-degrade, not a new failure mode), and the outputs-by-cell-id shape adaptation preserving only-insert-when-non-empty. Characterization tests over malformed fixtures first, then delete the serde parser.

3. **Retire the cfg(test) legacy loader** [L, ~245 LOC]. Right target, sequenced after the parser collapse (migrating tests earlier silently changes which parser they cover). The legacy loader is room-free while `prepare_initial_import` needs `&NotebookRoom`, so migrate in slices by test family: end-to-end behaviors to room-level materialization with tempdir rooms; pure-ingestion behaviors to direct tests of the shared helpers both loaders call. Not one L-sized pass.

4. **Reuse `prepared_runtime_sidecars` in staged import and reconciliation** [M, ~45 LOC]. Three field-for-field-identical mappings verified equivalent, including the stage_initial_import success-set argument. Call the helper before `prepared` is partially moved.

5. **One sync atomic-write core for the non-durable tier** [M, ~35 LOC]. Revision: one sync core (parent-create, perms-preserve, write/rename/cleanup, `io::Result`); `persist_notebook_bytes` becomes a bool-and-log adapter, persist's async `write_file_atomic` a thin wrapper, comments_store points at the core. Explicitly out of scope: `recovery.rs::replace_file_atomically` and `RealCheckpointIo::replace_temp` are the durable fsync tier and stay separate.

6. **Reuse `await_execution_terminal` in `run_all_and_wait`** [M, ~28 LOC]. Removes the duplicated terminal-status predicate. Revision: don't loop N executions with the default 500ms output-sync grace (tens of seconds of sleep on large notebooks); wait all-terminal with zero grace against the shared deadline, then one final grace pass on trailing empty-manifest executions. Decide KernelFailed (mark has_error, stop waiting) and Timeout mappings explicitly.

7. **Delete `AwaitedSessionReady`, staged** [L, ~45 LOC eventually]. Step 1 now: convert `create_notebook` (already holds a lease; projection fetch is proven in the connect paths). Step 2, separate PR gated on an end-to-end rejoin test: migrate `daemon_watch::rejoin`, which is the daemon-restart recovery path with no lease and no current e2e coverage. The variant and `NotebookSession::local` are only deletable after step 2.

8. **Delete `NotebookRoom::load_or_create`** [L, ~120 LOC]. ~140 lines of duplicated construction that skips the recovery-journal branch, so tests validate a room shape production cannot create. Corrected inventory: 10 call sites, not 4, in two migration patterns: legacy-persisted-doc fabrication moves to `NotebookDoc::save` directly; reload round-trip coverage moves to `new_fresh`'s untitled branch (the production analog). Mind the semantics gap: `load_or_create("test-nb")` fabricates a bogus file binding; migrated tests must pick the shape they mean.

9. **Delete the loading reset wrappers** [M, ~28 LOC]. `try_start_loading`/`finish_loading`/`reset_loading_for_test` force-reset through transitions production cannot take. Corrected inventory: 6 `finish_loading` sites across ~4 autosave-guard tests. The retry test must be rewritten via `retry_failed_claimed`/`RoomInitialLoadClaim`, not `complete_failed` + `begin()`, or Failed-stickiness silently changes its meaning.

10. **Remove the cfg(test) `apply_ipynb_changes` wrapper** [M, ~24 LOC]. Tests name whether they exercise the no-source-revision path explicitly. Requires promoting `apply_ipynb_changes_inner` to pub(crate) and moving the `.changed()` collapse to call sites.

11. **`TrustState::no_dependencies()` helper** [S, ~35 LOC]. Three verbatim 14-field literals become one explicit constructor in metadata.rs. Don't derive Default; the explicit name documents intent.

12. **Eviction generation-fence closure** [S, ~24 LOC]. One local closure in peer_eviction.rs capturing room + teardown_generation, called inside each `serialize_with` closure; the ShutdownKernel latch flip stays fused with its check in one critical section. Revision: registry.rs is untouched; `serialize_with` is the lock primitive and must not move.

13. **Named notebook-write-allowed predicate** [S, ~10 LOC]. Move the saved-path + four-arm settle match and its warn logs verbatim into one bool-returning async fn. Preserves the 120s timeout and branch partition by construction.

14. **Named readiness booleans in `NotebookSession::access`** [S]. Only with tests landing in the same change: the error-code priority chain has zero coverage today and a silent reorder is exactly what slips through. Hoisting to one status snapshot (the current chain re-reads status twice) is a stated decision, not a refactor accident.

15. **Reaper `remove_if` predicate extraction** [S]. Conditional: only as part of adding the missing unit tests for the connection-generation-bump and reservation-held abort legs. Without those tests it is net-additive churn on a high-subtlety invariant. If nobody signs up for the tests, skip.

16. **Demote the reaper's legacy .automerge flush, two steps** [M, riskiest item]. The file's existence gates the NotebookSync UUID reload branch for file-backed rooms too, so the naive demotion breaks UUID rejoin. Step 1: repoint the recoverability check (daemon.rs:2885) at journal discovery, with a test that a reaped room with a journal but no .automerge reloads by UUID. Step 2 only after: make the Step 2 flush best-effort post-barrier. Then the journal genuinely is the single durable-acceptance boundary the ADR claims.

### Dropped, with reasons

Recorded so these do not get re-proposed.

- **Consolidate temp-path helpers**: four helpers exist, not two; persist's callers have no save sequence; the distinct suffixes are crash diagnostics. More plumbing than the ~8 lines each copy costs.
- **Collapse bound-path predicates**: `is_primary_path` and `is_in_place_save` disagree on the untitled case by design, not drift. Unifying breaks baseline seeding on exactly the untitled-promotion binding save.
- **Move autosave-owner test seams**: already fully cfg(test)-gated with `_for_test` names; the production touchpoint must stay in the function regardless. Zero payoff in a load-bearing 2500-line file.
- **Narrow `InsertOutcome` visibility**: registry is a private module re-exported pub(crate); the type is already unreachable outside the crate. Keyword churn.
- **Schema structs as request parsers**: the arg helpers are deliberate claude-code#32524 workarounds (string booleans, stringified arrays) with pinned tests; serde replication costs more than it deletes and leaves the crate with two parsing styles.
- **Narrow the ADR conflict-preservation claim**: premise false. tests.rs:1486, 2948, and 3006 cover the divergent-fingerprint path end to end. The mapping-pass evidence gap is closed by those tests, not by an ADR edit.

## Verification roadmap

Surviving plans after cross-model review, ranked by value per effort. Duplicated plans were merged; the review's factual corrections (existing coverage the plans missed) are folded in. Dropped outright: the freeze-thaw stateright lock model and the whole-durability stateright model (hand-modeled abstractions that would drift from the code; stateright is a new dependency with one marginal payoff), the loading-generation cancellation matrix (already covered by tests.rs:930 and 973), and the composed slot/epoch proptest model (revisit only if the rollback test below surfaces gaps). The "skipped as adequate" boundary stands: sidecar best-effort, directory-chain durability, exported-heads ancestry, restart-identity recovery, and the conflict predicate all have direct tests; no new work there.

1. **Watcher ingest classifier: table tests + IN_ACCESS storm test** [M]. Guards the watcher re-ingest loop that has shipped twice. Extract `classify_watcher_observation` as a pure function (the three guards are already side-effect-free reads). Property: bytes match the baseline hash, or sha equals the manifest fingerprint, or (until deleted) inside the self-write window ⇒ Skip; unknown bytes ⇒ Ingest. Deterministic table first (self-write window, known disk hash, manifest fingerprint, both, unknown), then one integration storm test: 50 byte-identical debounced events ⇒ zero merges, zero sequence claims, zero journal appends, source generation unchanged. Plus the two tests the time-window deletion needs: event between journal commit and baseline install skips via the manifest fingerprint; event in the rename-to-commit window resolves via the pending-checkpoint comparison. First step: extract the predicate and write the storm test. Proptest generalization later. This gates consolidation item 1.

2. **Lifecycle transition-lock lint** [S]. Guards the generation-bump family at its root: a future transition method compiling cleanly without `lock_transition` gives torn dual-axis updates and duplicate leases. Source-scan lint test cloning the tokio_mutex_lint skeleton. Rule 1: any fn in lifecycle.rs mutating `source_tx`/`availability_tx`/`task_claimed`/staged/projection must call `lock_transition` before the first mutation (not necessarily first statement; guard clauses are fine). Rule 2: lifecycle.rs contains no doc-lock tokens, freezing the doc-before-transition lock order. First step: `crates/runtimed/tests/lifecycle_transition_lint.rs` with the token list and an allowlist for constructors and tests.

3. **Archive-replace ambiguous-rename fault matrix** [S]. Guards the no-active-journal state that downstream misreads as NotFound and mints a duplicate UUID (zombie-room/double-write class). The byte-compare ambiguity classifier is entirely untested. Property: under every fault cell the active journal path exists and scans to a valid latest record; wrote-then-errored classifies via byte-compare; the archive dir is complete once its rename commits. Tooling: plain test matrix through the existing `archive_and_replace_with` and `publish_archive_copy` seams. First step: extend the existing replacement-failure test into a table, starting with the wrote-then-errored cell.

4. **FileCheckpointCoordinator model proptest** [M]. Guards autosave starvation (a failed newer preparation permanently blocking older saves) and the stale-overwrite/cell-wipe class (older bytes replacing newer, then merged back as an external edit). Stateful proptest over reserve/bind/complete_with_callbacks/commit_existing_with with a scripted CheckpointIo. Revised oracles: barrier and checkpoint sequences monotone; reserve alone never supersedes; Blocked-before-replace leaves disk untouched and emits nothing; commit-fail-after-replace advances disk bytes but not the published checkpoint (the original oracle would have failed on correct behavior); Saved publishes both; fingerprint equals disk bytes when the checkpoint owns the path; a pre-replace fault on s2 never blocks a later s1 < s2 from reaching Saved. First step: add proptest as a runtimed dev-dependency (this plan establishes it for the repo), re-express the two existing interleaving tests as fixed op sequences to validate the oracles, then generate.

5. **Pending-checkpoint intent crash matrix, missing cells only** [S]. Guards the false-source-conflict lockout (room bricked on every restart) and the crash-window regenerate-over-saved-file path. Existing coverage is four tests (two unit, two integration), not two; target the genuinely uncovered cells: abort-write-failed, commit-append-failed after visible replace, file-missing-at-restart, reconciliation intent with source_generation, idempotent double-resolve. Property: `resolve_recovered_file_checkpoint` returns exactly the trichotomy table (intended ⇒ Committed with advanced heads; previous ⇒ pending cleared, no advance; third ⇒ SourceConflict with pending preserved), and the recovery union is bit-identical across resolution. First step: seed the table harness with the four existing tests as validated rows.

6. **Journal byte-model proptest** [M]. Guards acknowledged-data loss from a misclassified tail: a wrong `last_valid_end` drops the last acked batch or truncates corruption evidence. Scoped to the byte model only (sidecar and directory-chain already have direct tests). Property: for any journal and truncation length L, scan recovers exactly the last record whose full extent fits; a byte flip strictly inside a valid record makes append refuse with CorruptTailRequiresReconciliation and never shortens the file; torn-tail repair truncates to exactly the last valid extent; any external length/mtime change forces the cursor rescan. First step: proptest strategies over `encode_record`-built journals, porting the existing torn-tail test as the first shrinkable case.

7. **RoomDurability model proptest with crash-equivalence oracle** [M]. Guards the generation-bump shape applied to durability: a stale Daemon snapshot landing over newer acknowledged peer commits. Sync ops only (freeze/thaw interleaving is out of scope per review): commit_snapshot with stale candidates, commit_peer_changes with forked actors, commit_staged_source, checkpoint prepare/commit/abort/resolve, journal-append faults. Oracles: everything acknowledged replays from `latest_record()` (the executable statement of journal-before-ack); durable/exported heads contained in the snapshot; manifest sequence increases only on Committed; journal errors set degraded and `await_durable` fails fast; commit_file_checkpoint never clears a pre-existing degraded reason (the asymmetry currently held by a comment). First step: the forked-Automerge trace generator, then the existing stale-snapshot test replayed through the model oracles.

8. **RoomLifecycle state-machine tests, real type** [M]. Guards the generation-bump family directly: stale gen-N completions installing artifacts over gen N+1, reopens erasing a Failed room's retry policy, Loading generations with no owner. Deterministic stale-generation tests against the existing bool-returning methods first; then proptest driving the real type with ops carrying generation tags in {g, g-1, g+1}. Invariants: at most one Started per generation; Ready/Failed stickiness; mismatched-tag completions return false with the full observable snapshot bitwise unchanged; terminal transitions reset the claim; the capability lattice (mutate ⇒ Ready with projection). The `complete_reconciliation` debug_assert-to-fallible API change is a separate deliberate decision, made only if the generator needs that op.

9. **Primary baseline gate + superseded continuation** [S]. Guards the permanent autosave wedge (disk_content_diverged stuck true) and the historical cell-wipe class (older Save As installing a watcher baseline for the wrong file). Deterministic permutation tests on `note_primary_save_baseline`: out-of-order and duplicate sequences, asserting the maximal accepted call owns the whole four-field tuple, never a mix. One continuation-race test via the checkpoint callback seams: s2 commits first, s1 must fail the manifest triple-check and rebind nothing. Proptest second, not a blocker.

10. **Peer rollback byte-compare + multi-frame faults + narrowed mutation lint** [M]. Guards silent acked-data loss that Automerge convergence then propagates as authoritative absence. The single-fault rollback test exists (tests.rs:2650); extend it to byte-compare the encoded `sync::State` before and after failure (the desync half has zero coverage), then a multi-frame harness failing the Nth change-bearing frame and asserting restart replays exactly frames 1..N-1. The lint: allowlisted scan over NotebookDoc mutation paths specifically, not every `changed_tx` send in the crate (approve_project_environment and hosted_bridge would false-positive). This is what fails the build when a future mutation call site skips the durability boundary.

11. **`install_activated_session` rollback test** [S]. Guards wrong-document targeting: a superseded connect publishing its session over the notebook the user just switched to. #2088-adjacent surface. Deterministic tokio test: stub the connect future with a oneshot, supersede mid-flight via `begin(B)`, assert `mark_installed` fails, the previous slot occupant is restored, and the result is session_superseded. First and only immediately-justified artifact for this cluster.

12. **Staged-import replay idempotence, deterministic** [M]. Guards the history-fork class: a retry that regenerates changes gives peers permanently divergent hashes for identical content. Overlaps #4015 (metadata-apply idempotence); land these as additional rows in that issue's harness rather than a second one. Rows: publish prefixes 0..n then re-run with the retained artifact plus an injected peer change, asserting final change-hash set equals staged ∪ peer hashes with no regenerated hashes; disk-fingerprint conflict gate applies zero batches; finalize containment fails with source_degraded on a missing hash. Proptest only if the deterministic helper stays cheap.

13. **Destructive-teardown pause hook, first site only** [L]. Guards the kernel-shot-mid-reconnect class (a fast reconnect that saw has_kernel=true loses its live kernel; the visible symptom family of the workstation silent-launch bug). Land one cfg(test) pause hook immediately before the ShutdownKernel revalidation, drive a real connect during the pause, assert teardown aborts, the destructive latch stays false, and no ShutdownKernel RPC fires. Add the other two sites only after the first hook proves non-invasive.

## Suggested sequencing

1. **Sweep PR**: item 1 of delete-now (comments, allows, renames, ADR fixes), folded into #4018.
2. **Lifecycle transition lint** (verification 2): cheapest harness, guards the generation-bump root, no dependencies.
3. **Dead-surface deletion train**: journal-durability deletions (archive API, compact + append narrowing), checkpoint wrappers and event bus, mcp-client dead branches, protocol/TS deletions (SaveError, parity tests, GuardedNotebookProvenance, fail-closed defaults), micro-deletions. Mechanical, verified, no ordering constraints beyond compact-with-append.
4. **Watcher classifier harness, then the time-window deletion** (verification 1, then consolidation 1): the three tests land first, the window and `last_self_write` die in the same series. This closes the twice-shipped incident class with evidence instead of a comment.
5. **FileCheckpointCoordinator model** (verification 4): establishes proptest as a dev-dependency and covers the autosave-starvation and cell-wipe oracles the deletion train's test rewrites lean on.
6. **RoomInitialLoad legacy channel + frame-drain cleanup** (delete-now 6 and 12): one focused PR on the room-substrate seams, full suite run after.
7. **Consolidations in dependency order**: sidecar mapping reuse, atomic-write core, run_all wait; then parser collapse followed by the sliced legacy-loader retirement.
8. **Staged risky items last**: AwaitedSessionReady step 1 (create path only), then the .automerge flush demotion behind its journal-discovery repoint and UUID-rejoin test.
